Modelsim 2019.2笔记

从代码到波形

如何从代码文件到波形图

对于Verilog：

1 写好top和testbench

2 写一个sim.do

sim.do的样例如下：

source: D:\Lessons\数字集成电路I\2020VLSI\e1_adder

多文件版本的sim.do：

source: D:\Lessons\数字集成电路I\RISCCPU\project\risk_cpu\cpu

3 进入相关目录然后do sim.do

更加一般的方法，例如对于System Verilog文件：

1 写好top和tesebench

2 参考modelsim详细使用教程，但是这个文件被垃圾百度文库设成了VIP才能阅读全文。简单来说，首先建立work这个library：File->New->Library。如果已经有work目录，就直接新建：File->New->project，然后就很自然了，add existing files，然后compile all，切到Library选项卡下，右键testbench的文件选择simulate即可。如下图：

2.5 如何看compile_all的报错？直接双击就行了

3 【10.4】vsim -gui work.testbench 【2019.2】vsim -voptargs=+acc work.top_testbench

4 add wave *

5 调整run length，如改成1 us。注意自己testbench开头写的timecale。

6 输入run启动仿真

7 仿真后quit -sim或restart

del波形直接在wave - default里面delete就行了

觉得某个波形前缀太长了的话下面有个滑动条可以拉一拉

wave窗口左下角还有个小按钮可以切换leaf name和full name

按G可以go to time

快捷键

2019.2下，不能简单的用vsim -gui work.testbench启动仿真

而是需要vsim -voptargs=+acc work.testbench（即在Library中右键testbench.sv选择第一个simulate）

点击sim - Default窗口的Instance可以切换目录树的排序方式

下面是10.4版本的快捷键，新版本可能有变化

vlog：针对verilog

vcom：针对VHDL

针对sv的到底是啥啊！！！

Alt+C+A：编译全部

Alt+S+S：运行仿真

run -all：仿真命令

vsim -gui lib.modelname：运行仿真

add wave *：把所有top模块的输入输出变量加到wave里面

在sim-Default Instance中选中所需模块直接Ctrl + W即可将该模块所有信号加入Wave

Tools - Edit Preferences可以改字体大小

cd E:/study/Fundamentals_of_IC_Design/RISCCPU/project/risk_cpu/counter?

add wave -r /*：把所有中间变量都加到仿真里面去

专门把某个中间变量添加到仿真：

add wave sim:/仿真模块的名称/仿真模块中所实例化的模块的实例化名称/所需中间变量

如：add wave sim:/npu_nl_relu_tb/nnr/relu_result

add wave sim:/npu_nl_relu_tb/nnr/relun_result

用命令可能还不如用图形界面快。用图形界面的方法是：

在sim-default窗口中，点击所需变量所在的模块，该模块中所有变量立即被加入到Objects中，然后右键或Ctrl+W添加波形，重新仿真。

所谓的sim-default窗口，就是View中的Structure窗口

用指令也可以，比如add wave -position insertpoint sim:/testbench/blkbfs/*

或者：add wave -position insertpoint sim:/testbench/*

这些指令不会递归。

如果要删除Wave，那么就是直接选中按Delete即可。

quit -sim：退出仿真

三操作：

Alt +C+A ：编译全部

restart：重启仿真

run -all ：仿真开启。不要选用run -all，还是建议使用run循环…用来调试….

在wave窗口中：

i：放大

o：缩小

f：全部显示

错误和警告

非常莫名其妙的东西：set PrefMain(updateRate) 797n
