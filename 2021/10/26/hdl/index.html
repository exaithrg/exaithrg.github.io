锁存器问题

verilog入门经验（一） always块使用

任何在always块内被赋值的变量都必须是寄存器型（reg）。即<=或=左边的信号，必须是reg型，<=或=右边的信号可以是reg型也可以是wire型。

端口声明中被声明为input或inout型的端口，只能被定义为线网型（wire）；被声明为output型的端口，则可以被定义为线网型（wire）或者寄存器型（reg）。如果不定义，则默认为线网型（wire）。

always的敏感列表中可以同时包括多个电平敏感事件，也可以同时包括多个边沿敏感事件，但不能同时有电平和边沿敏感事件。另外，敏感列表中，同时包括一个信号的上升沿敏感事件和下降沿敏感事件也是不允许的，因为这两个事件可以合并为一个电平事件。

问题1：0.5的定点补码是？

有符号定点小数的补码表示

根据这个，则0.5原码0.1000，反码0.1000，补码0.1000【=bmwidth(0.5*2^1)】

问题2：-0.5的定点补码是？

原码1.1000，反码1.0111，补码1.1000【与原码一致，巧合】【=bmwidth(-0.52^1)=*1，小数点自行约定】

问题2：-0.125的定点补码是？

原码1.0010，反码1.1101，补码1.1110【=bmwidth(-0.1252^3)=*1，小数点自行约定】

问题3：2.25的定点补码是？

原码010.01，反码010.01，补码010.01【=bmwidth(2)+2】【=bmwidth(2.25*2^2)】

问题4：1.25的定点补码是？

原码01.01，反码01.01，补码01.01【=bmwidth(1)+2】【=bmwidth(1.25*2^2)】

问题5：-2的定点补码是？

原码110，反码101，补码110【=bmwidth(-2)=2】【=bmwidth(-2*2^0)】

问题6：-1.5的定点补码是？

原码11.100，反码10.011，补码10.100【!=bmwidth(-1)+1=2。警告：至少需要3位。】【=bmwidth(-1.5*2^1)】

问题7：-2.125的定点补码是？

原码110.00100，反码101.11011，补码101.11100【!=bmwidth(-2)+3=5。警告：至少需要6位。】

问题8：-2.75的定点补码是？

原码110.1100，反码101.0011，补码101.0100【!=bmwidth(-2)+2=4。警告：至少需要5位。】

综上，负数补码不能单看整数部分最少多少位、小数部分最少多少位，加起来得到最终最少多少位。这一点在极限范围设计时非常重要。假设已知小数位数为fw，最佳解决方案是按bmwidth(fix(x*2^fw)来决定存储位数，必不会错。

因此针对小数的bmwidth函数可以用function binwidth = bmw_frac(fracnum,fracwidth)的形式，并加入decnum = fix(fracnum.*2.^fracwidth);【注意.*和.^】。

最终实现了两个非常有用的函数：bmw_int(decnum)和bmw_frac(fracnum,fracwidth)，尤其bmw_frac特别好使。另外宝马打钱！

bmw_frac的作用应该改一下，改成给出能够表示整数或小数x（如x=-1.54296875）的最短位数补码的位数。返回值应该是[位数，误差]。存在一个上限，比如64位补码也表示不了，就给出误差。

这几个函数还可以大量扩展，命名也得多注意。

为什么Windows下1TB硬盘只有931GB？

硬件厂商说我这个盘是1TB，实际上就是1TB。但在windows下，显示虽然是TB，但是实际上是按TiB算的，也就是说1TB=0.9095TiB，而1TiB=1024GiB，所以Windows下就会显示931.328GiB（GB）了。

Windows显示5.44PB，实际上是5.44PiB，1PiB=1.1259PB，所以对于硬件厂商来说就是6.124896PB的硬盘。

硬件厂商和Windows所说的B都是8位，但是对K的定义有区别，因此导致后面的M G T都有区别。

需要进行两步换算：

首先1TB实际上是1TB，但是1GB实际上是1GiB。即问：1TB = 多少GiB？

1TB = 0.9095TiB

1TiB = 1024 GiB

1TB = 0.9095 * 1024 = 931.3GiB

或者

1TB = 1000GB

1GB = 0.9313GiB

1TB = 1000 * 0.9313 = 931.3GiB

因此在Windows系统下所有的KB，MB，GB全都指的是KiB MiB GiB

2的幂次

2^1 = 2 2^2 = 4 2^3 = 8 2^4 = 16 2^5 = 32 2^6 = 64

2^7 = 128 2^8 = 256 2^9 = 512 2^10 = 1024 2^11 = 2048

2^12 = 4096 2^13 = 8192 2^14 = 16384 2^15 = 32768

2^16 = 64 * 2^10 = 65536 2^32 = 4 2^10 2^10 * 2^10

2^64 = 16 2^60 2^10B = 1KiB 2^10 2^10B = 1MiB

2^10 2^10 2^10 = 1GiB 2^32B = 4GiB

32位电脑最大支持4GiB内存，这是因为内存每个地址对应1B，而32位最多能表示4Gi个地址。

64位电脑最大支持16EiB内存，也就是16384PiB内存，也就是16777216TiB内存
