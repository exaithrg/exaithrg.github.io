DB60M34A6435V1毫米波雷达核心板说明.pdf

此核心板是没有本公司提供的特制算法程序的，但是其内部的程序可以参考TI的官方例程（具有诸多的参考例程），不但可以进行例程演示，还可以通过CCS查看例程的源代码甚至在此基础上做进一步的修改开发，具体获得例程的方法请查阅TI官网。

这个核心板是一个最小的雷达系统，如果想要跑TI的例程，需要配套本公司的DBMMDEBUG调试板，这个调试板通过排针与核心板连接，两个板子组成一套基本上可以跑TI的所有例程，这个DBMMDEBUG调试板是XDS110仿真器，可以直接通过CCS进行程序的在线仿真，所以这里也不需要再购买TI的其他JTAG仿真器。

这个核心板为了减小体积以及降低成本，内部并没有将LVDS高速数据接口引出来，而引出来的串行接口如SPI、UART、CAN等，其传输速度较低不能满足实时ADC原始数据的输出，所以这个核心板不能实时获得ADC的原始数据，但是如果您对原始ADC数据的实时性没有要求，可以使用此核心板的串行接口慢慢打印原始ADC数据，这里举个使用UART打印原始ADC数据的例子，通过更改IWR6843内部程序，可以将ADC转换的数据存储到内部的L3 RAM中，这里假设ADC数据占用了全部的L3 RAM即768KB，UART波特率为921600bps，将所有ADC数据传输出来所花费的时间大致为768102410/921600≈8.54s（这里乘以10是因为串口发送有起始和停止位）。如果使用SPI进行传输的话，这里假设使用SPI的CLK为50MHz，打印所有ADC数据所花时间大致为76810248/50000000≈0.126s。所以只要对ADC数据的实时性没有要求的话，也是可以通过更改内部程序以及接口，实现原始ADC数据的输出。

客户可以自己根据供电的电源、通信接口自行设计底板（替代XDS110调试板），具体的接口说明后文会详细解释，用户根据核心板的尺寸、接口（使用UART、SPI或者是CAN等都可以）、系统电源（12V、5V）等设计好底板后，可以直接将核心板装上使用，注意一些接口以及电源一定符合核心板要求，如果可以的话本公司可以帮忙检查设计的底板。

为了避免核心板损坏，请严格注意以下几点：其一就是供电电压，一定不能使用超限的电源进行供电；其二是插拔核心板时一定要对系统断电；其三插拔核心板时要对齐排针和排母，不能错位插入，细心检查后再给系统上电（因为此核心板使用了防呆的排针，所以排针排母不对齐插入的现象应该不会再发生）；其四是测试过程中尽量不要用手触碰天线，可以的话请使用放电手环。

DB60M34A6435V1毫米波雷达核心板内部已经处理好了SOP0、SOP1、SOP2的默认状态，其中SOP0=1，SOP1=0，SOP2=0，其中SOP2通过排针引出，这里只能改变SOP2的状态，从而改变相应的运行模式。SOP2=0，雷达从Flash读取程序并正常运行；SOP2=1，UART向Flash烧程序。

iwr6843.pdf

IWR6843：1.75MB，分为 MSS 程序 RAM (512KB)、MSS 数据 RAM (192KB)、DSP L1 RAM (64KB) 和 L2 RAM (256KB) 以及 L3 雷达数据立方体 RAM (768KB)
